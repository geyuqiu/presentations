common failures

  • begins coding too early before completing data structure brainstorming and thinking about all of the steps along the way

Recommended Steps

  1. explain a brute force asap (not code it), slow but works
  2. stating time and space complexity for brute force approach
  3. Optimize
  4. think about every data structure and variable you will use
  5. apply tdd if possible, but also follow your instinct, maybe reverse engineer it !
  6. explain your approach on a white paper
